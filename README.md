# AEGIS — Academic Evidence-Guided Information System

AEGIS is a local-first Retrieval-Augmented Generation (RAG) assistant combining a FastAPI backend, a React frontend, local ChromaDB vector store for embeddings, Supabase for chat history, and an Ollama-managed LLM for generation.

**Key features**
- Local vector store (ChromaDB) for fast retrieval
- Pluggable LLM via Ollama (default set to `gemma4:e4b`)
- React + Vite frontend for interactive chat UI
- Integrations: Wikipedia, arXiv search, PDF/text ingestion

## Quickstart (Linux)

Prerequisites

- Python 3.10+ (3.14 tested)
- Node.js 18+ (use `nvm` to install safely)
- Ollama running locally if you want to use local LLMs (`ollama serve`)

1. Create and activate a virtual environment (do NOT install globally):

```bash
python3 -m venv venv
source venv/bin/activate
```

2. Install Python dependencies inside the venv:

```bash
pip install -r requirements.txt
```

Note: The repo includes a minimal `requirements.txt`. If you need GPU support for `torch`, install the appropriate wheel from PyTorch rather than the defaults.

3. Frontend (requires modern Node):

```bash
# Install nvm if needed, then install node 20:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20

cd frontend
npm install
npm run dev
```

If you see a Vite / `node:perf_hooks` error, your Node version is too old; upgrade via `nvm` as shown above.

4. Environment and Supabase

- The project expects Supabase credentials if you want to persist chat history. Create a `.env` and add `SUPABASE_URL` and `SUPABASE_KEY` or edit `supabase_client.py` directly if you prefer. Avoid committing secrets.

5. Start the backend (from repo root, with `venv` activated):

```bash
uvicorn server:app --host 0.0.0.0 --port 8000
```

The API will run on `http://localhost:8000` and the frontend defaults to `http://localhost:5173` (Vite).

## Model configuration (Ollama)

This project uses Ollama as its local LLM runtime. The Python interface file is `models/ollama_llm.py` and the default model is set to `gemma4:e4b`. To change the model, edit the `DEFAULT_MODEL` variable in that file or override the model parameter when calling the query function.

If you run Ollama locally, ensure `ollama serve` is running and the model `gemma4:e4b` is available.
